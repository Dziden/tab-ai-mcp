# Архитектура ТАБ:Библиотека ИИ (tab_mcp)

## Компоненты

| Компонент | Описание | Размещение |
|---|---|---|
| **БИИ** | ТАБ:Библиотека ИИ — установлена внутри 1С, предоставляет HTTP-сервис для записи данных | CPU, внутри 1С |
| **База 1С** | 1С:Предприятие — источник данных, OData REST API и HTTP-сервис БИИ | — |
| **tab_mcp** | MCP-коннектор к 1С. Два инструмента: чтение через OData, запись через tab_ss | CPU |
| **tab_ss** | Семантический поиск, прогнозирование, аномалии, RAG, fine-tuning. Встроенный LLM при наличии GPU. Роутит запросы в БИИ: напрямую (on-prem) или через Элемент (cloud) | CPU / GPU |
| **tab_ca** | Мультиагент LLM | GPU |
| **Элемент** | Биллинг и тарификация. В облачном режиме все запросы к БИИ проходят через него | CPU |
| **LLM провайдеры** | GigaChat, OpenAI, DeepSeek и др. | Cloud / GPU |

---

## Два пути взаимодействия с 1С

### Путь 1 — Чтение через OData

```
tab_ss → tab_mcp → OData → База 1С
```

- OData — стандартный REST API 1С, не требует БИИ
- Используется для автоматического получения данных (базовая таблица для семантического поиска, RAG, fine-tuning)
- **Если OData недоступен** — этот путь не используется. Прикладной разработчик сам делает выборку из 1С и отправляет данные в tab_ss через HTTP (пример: функция `ПоискАномалий`)

### Путь 2 — Запись через БИИ HTTP

**On-prem:**
```
tab_ss → tab_mcp → tab_ss → БИИ (HTTP) → База 1С
```

**Cloud:**
```
tab_ss → tab_mcp → tab_ss → Элемент → БИИ (HTTP) → База 1С
```

- БИИ предоставляет HTTP-сервис с XDTO EnterpriseData загрузчиком — всегда доступен
- Механизм записи: OData формирует объект + XDTO EnterpriseData формирует объект → сравниваем качество заполнения полей → мержим лучшее → записываем
- Пользователь настраивает только один адрес — tab_ss. Роутинг через Элемент (cloud) или напрямую (on-prem) — внутренняя логика tab_ss
- Совместимость on-prem и cloud обеспечена: интерфейс одинаковый, путь разный

## Режимы работы БИИ

Не все функции библиотеки работают с данными 1С:

| Функция | Данные 1С нужны | Путь |
|---|---|---|
| Переводчик текста | Нет | БИИ → tab_ss напрямую |
| OCR (распознавание изображений) | Нет | БИИ → tab_ss напрямую |
| Проверка подписи документа | Нет | БИИ → tab_ss напрямую |
| Семантический поиск | Да (через OData) | БИИ → tab_ss → tab_mcp → OData |
| Поиск аномалий | Да (разработчик сам отбирает) | БИИ → tab_ss (данные в теле запроса) |
| Прогноз динамики | Да (разработчик сам отбирает) | БИИ → tab_ss (данные в теле запроса) |
| Запись результатов в 1С | Да | tab_mcp → tab_ss → [Элемент →] БИИ HTTP |

## Инструменты tab_mcp

### `read_1c`

Читает данные из 1С через OData.

```
Вход:  entity_type, filter?, select?, top?, skip?
Выход: список объектов
```

- Используется tab_ss как "базовая таблица" для индексации в векторную БД (dense + sparse векторы)
- Поддерживает пагинацию для батчевого чтения
- Если OData недоступен — не используется (данные приходят напрямую от разработчика)

### `write_1c`

Записывает данные в 1С через HTTP-сервис БИИ.

```
Вход:  entity_type, data (объект или список)
Выход: результат записи
```

- tab_mcp передаёт данные в tab_ss
- tab_ss доставляет в БИИ (on-prem: напрямую, cloud: через Элемент)
- Механизм записи в БИИ: XDTO EnterpriseData загрузчик (create или update автоматически)
- Качество данных: OData-структура и XDTO-структура дополняют друг друга перед записью

---

## Биллинг (Cloud)

В облачном режиме **все запросы** проходят через Элемент:
- БИИ → Элемент → tab_ss (входящие запросы от 1С)
- tab_ss → Элемент → БИИ (запись результатов в 1С)

Элемент фиксирует каждый запрос и списывает стоимость согласно тарифу.

## Схема (текстовая)

```
                    [LLM провайдеры]
                     ↑           ↑
                  cloud        cloud
                     |           |
[tab_ca GPU] ←─ onprem ─→ [tab_ss CPU] ←─── [tab_mcp CPU]
                                │    \              │
                             onprem  cloud        OData
                                │      \            │
                           [БИИ/1С] ←─ [Элемент] [База 1С]
                                ↑
                              БИИ HTTP
                           (XDTO загрузчик)
```
